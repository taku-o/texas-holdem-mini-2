# 実装タスクリスト

## 受け入れ基準（requirements.mdより）

> **要件 1**: Playwright環境構築（1.1〜1.5）
> **要件 2**: 初期画面のE2Eテスト + スクリーンショットベースライン（2.1〜2.4）
> **要件 3**: ゲーム画面レイアウトのE2Eテスト + スクリーンショットベースライン（3.1〜3.7）
> **要件 4**: カード表示のE2Eテスト（4.1〜4.3）
> **要件 5**: プレイヤー操作のE2Eテスト（5.1〜5.5）
> **要件 6**: ゲームフロー統合E2Eテスト（6.1〜6.6）

---

- [x] 1. Playwright環境構築
- [x] 1.1 Playwrightのインストールと設定ファイル作成
  - `@playwright/test` をdevDependencyとしてインストールする
  - Playwrightのブラウザバイナリをインストールする（Chromiumのみ）
  - Playwright設定ファイルを作成し、Vite devサーバーの自動起動を設定する
  - ビューポートサイズを固定する（BoundingBox検証の一貫性確保）
  - スクリーンショット比較の閾値設定を含める（idle画面は厳密比較、ゲーム画面は高めの `maxDiffPixelRatio`）
  - _Requirements: 1.1, 1.2, 1.3_
  - _Contracts: PlaywrightConfig_

- [x] 1.2 package.jsonへのテストスクリプト追加
  - E2Eテスト実行用の `test:e2e` スクリプトを追加する（`playwright test`）
  - ベースライン更新用の `test:e2e:update` スクリプトを追加する（`playwright test --update-snapshots`）
  - テストディレクトリ `e2e/` を作成する
  - _Requirements: 1.4, 1.5_
  - _Contracts: PackageScripts_

- [x] 2. 既存コンポーネントへのテスト用セレクター追加
- [x] 2.1 テーブルとアプリケーションコンポーネントへの `data-testid` 追加
  - テーブルコンテナにテスト用識別子を追加する
  - ポット表示エリアにテスト用識別子を追加する
  - コミュニティカードコンテナにテスト用識別子を追加する
  - アクションログコンテナにテスト用識別子を追加する
  - 既存の表示や動作に一切影響がないことを確認する
  - _Requirements: 3.2, 3.6, 6.1, 6.2, 6.3, 6.4, 6.5, 6.6_
  - _Contracts: TestIdAttributes_

- [x] 2.2 (P) プレイヤーコンポーネントへの `data-testid` 追加
  - プレイヤーコンテナに動的なテスト用識別子を追加する（プレイヤーIDを含む）
  - カードコンテナにテスト用識別子を追加する
  - ロールバッジにテスト用識別子を追加する
  - アクションバッジにテスト用識別子を追加する
  - 既存の表示や動作に一切影響がないことを確認する
  - _Requirements: 3.1, 3.4, 3.5, 4.1, 4.2, 5.3_
  - _Contracts: TestIdAttributes_

- [x] 2.3 (P) コントロールコンポーネントへの `data-testid` 追加
  - コントロールコンテナにテスト用識別子を追加する
  - 既存の表示や動作に一切影響がないことを確認する
  - _Requirements: 5.1, 5.2_
  - _Contracts: TestIdAttributes_

- [x] 3. 初期画面のE2Eテスト + スクリーンショットベースライン
- [x] 3.1 初期画面の表示検証テスト作成
  - 初期画面（idle画面）のテストファイルを作成する
  - タイトル「Texas Hold'em」の表示をテキスト一致で検証する
  - 「Start Game」ボタンの存在を検証する（要素数 = 1）
  - ボタンのBoundingBoxを取得し、幅 > 0、高さ > 0、画面内に収まっていることを検証する
  - 初期画面のスクリーンショットベースラインを `toHaveScreenshot('idle-screen.png')` で保存する
  - `npm run test:e2e` でテストがパスすることを確認する
  - タスク終了時に `npm run test:e2e:update` を実行してスクリーンショットベースラインを更新する
  - _Requirements: 2.1, 2.2, 2.3, 2.4_
  - _Contracts: IdleScreenSpec_

- [ ] 4. ゲーム画面レイアウトのE2Eテスト + スクリーンショットベースライン
- [ ] 4.1 ゲーム画面のレイアウト検証テスト作成
  - ゲーム画面レイアウトのテストファイルを作成する
  - ゲーム開始後にプレイヤー情報要素が5つ表示されることを検証する
  - ポット金額のテキストが `$` を含み数値を含むことを検証する
  - Fold / Check(Call) / Raise ボタンの存在を検証する
  - ロールバッジ（D / SB / BB）の表示テキストを検証する
  - 各プレイヤーのBoundingBoxを取得し、画面内に収まっていることを検証する
  - テーブル要素のBoundingBoxを取得し、幅・高さが最低値以上であることを検証する
  - ゲーム画面のスクリーンショットベースラインを保存する
  - `npm run test:e2e` でテストがパスすることを確認する
  - タスク終了時に `npm run test:e2e:update` を実行してスクリーンショットベースラインを更新する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7_
  - _Contracts: GameLayoutSpec_

- [ ] 5. カード表示のE2Eテスト
- [ ] 5.1 (P) カード表示の検証テスト作成
  - タスク2完了後にタスク6.1と並列実行可能
  - カード表示のテストファイルを作成する
  - ヒューマンプレイヤーのカードにランク文字（2〜A）が2つ表示されていることを検証する
  - CPUプレイヤーのカードにランク文字が表示されていない（裏面）ことを検証する
  - カード要素のBoundingBoxを取得し、幅・高さが期待範囲内であることを検証する
  - ヒューマンプレイヤーの特定は「You (Player)」テキストを含むプレイヤーをスキャンして行う
  - `npm run test:e2e` でテストがパスすることを確認する
  - _Requirements: 4.1, 4.2, 4.3_
  - _Contracts: CardDisplaySpec_

- [ ] 6. プレイヤー操作のE2Eテスト
- [ ] 6.1 (P) プレイヤー操作の検証テスト作成
  - タスク2完了後にタスク5.1と並列実行可能
  - プレイヤー操作のテストファイルを作成する
  - ヒューマンプレイヤーのターン時にコントロールの `opacity` CSS値が0.5でないことを検証する
  - 非ターン時に `opacity` = 0.5 または `pointer-events` = `none` であることを検証する
  - Foldボタンクリック後にプレイヤーのアクションバッジテキストが "FOLD"（uppercase）であることを検証する
  - Check/Callボタンのテキストが状況に応じて正しいことを検証する（callAmount > 0 なら "Call $数値"、それ以外なら "Check"）
  - Raiseボタンの金額テキストに数値が含まれることを検証する
  - ヒューマンターンの待機はコントロールのCSS状態変化を `expect` の `timeout` で待機する
  - `npm run test:e2e` でテストがパスすることを確認する
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5_
  - _Contracts: PlayerControlsSpec_

- [ ] 7. ゲームフロー統合E2Eテスト
- [ ] 7.1 ゲームフロー統合テスト作成
  - ゲームフロー統合のテストファイルを作成する
  - pre-flopフェーズでコミュニティカード表示枚数 = 0 を検証する
  - flopフェーズに進んだ後にコミュニティカード表示枚数 = 3 を検証する
  - turnフェーズに進んだ後にコミュニティカード表示枚数 = 4 を検証する
  - riverフェーズに進んだ後にコミュニティカード表示枚数 = 5 を検証する
  - ゲーム進行中にアクションログ要素数 ≥ 1 を検証する
  - ショーダウン終了後にポット金額が $0（分配済み）であることを検証する
  - 各フェーズでヒューマンプレイヤーのアクション（CallまたはCheck）を実行してフェーズを進行させる
  - フェーズ遷移の検出はテーブル下部のフェーズ名テキストの変化とコミュニティカード数で行う
  - テストタイムアウトを十分に確保する（CPU行動の遅延を考慮）
  - `npm run test:e2e` で全テストがパスすることを確認する
  - タスク終了時に `npm run test:e2e:update` を実行してスクリーンショットベースラインを更新する
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5, 6.6_
  - _Contracts: GameFlowSpec_
