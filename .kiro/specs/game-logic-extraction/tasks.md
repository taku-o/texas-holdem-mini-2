## タスクリスト

- [ ] 1. ゲームロジック純粋関数の抽出
- [ ] 1.1 (P) 次のアクティブプレイヤー取得関数を作成する
  - 現在のプレイヤーインデックスとプレイヤー配列を受け取り、次にアクションするプレイヤーのインデックスを返す純粋関数を作成する
  - フォールド済みのプレイヤーとチップが0のプレイヤーをスキップする
  - プレイヤー数のハードコード（5）を配列の長さに基づく計算に置き換える
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [ ] 1.2 (P) ベッティングラウンド終了判定関数を作成する
  - プレイヤー配列と現在のベット額を受け取り、ラウンドが終了したかどうかを判定する純粋関数を作成する
  - 全員が行動済みでベット額が一致している場合にtrueを返す
  - 未行動者がいる場合やベット額が一致していない場合にfalseを返す
  - フォールドしていないアクティブプレイヤーが1人だけの場合にtrueを返す
  - オールインプレイヤー（チップ0）はアクション不要として扱う
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6_

- [ ] 1.3 (P) ブラインドポジション計算関数を作成する
  - プレイヤー配列と前回のディーラーインデックスを受け取り、次のハンドのdealer・SB・BB・UTGの各インデックスを返す純粋関数を作成する
  - 初回（dealerIndexが-1）の場合は、インデックス0から次のアクティブプレイヤーをディーラーとして計算を開始する
  - 非アクティブプレイヤーをスキップして正しいポジションを計算する
  - プレイヤー数のハードコード（5）を配列の長さに基づく計算に置き換える
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [ ] 1.4 (P) アクション適用関数を作成する
  - プレイヤー配列・対象プレイヤーインデックス・アクション種別・金額・ポット・現在のベット額を受け取り、更新後のプレイヤー配列・ポット・ベット額・ログメッセージを返す純粋関数を作成する
  - Fold: 対象プレイヤーのアクションをfoldに設定する
  - Call: 対象プレイヤーのチップからコール額を減算し、ポットに加算する。チップが0になった場合はオールインとする
  - Raise: 最低レイズ額（現在のベット額の2倍）以上のベットを行い、ポットに加算する。チップが0になった場合はオールインとする
  - 入力データを変更せず、新しいオブジェクトを返す
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_

- [ ] 1.5 (P) 勝者判定関数を作成する
  - フォールドしていないアクティブプレイヤーの配列とコミュニティカードを受け取り、最強ハンドを持つプレイヤーのID・名前・役名を返す純粋関数を作成する
  - 既存のハンド評価機能を利用して各プレイヤーのハンドを評価する
  - _Requirements: 5.1, 5.2_

- [ ] 2. ゲームエンジンフックのリファクタリング
- [ ] 2.1 型定義と定数の移動、およびゲームロジック関数の統合
  - ゲーム状態の型定義とゲーム定数をゲームロジックモジュールに移動する
  - ゲームエンジンフックからは移動した型・定数をre-exportして後方互換性を維持する
  - ゲームエンジンフック内のローカル関数呼び出しを、抽出した純粋関数の呼び出しに置き換える
  - ブラインド計算のインラインロジックをブラインド計算関数の呼び出しに置き換える
  - アクション処理の純粋計算部分をアクション適用関数の呼び出しに置き換える（副作用はフック内に残す）
  - ショーダウン時の勝者判定ロジックを勝者判定関数の呼び出しに置き換える
  - ゲームエンジンフックの外部APIインターフェースは変更しない
  - _Requirements: 1.5, 2.7, 3.5, 4.6, 5.3, 6.1, 6.2, 6.3_

- [ ] 3. 抽出した関数の単体テスト
- [ ] 3.1 getNextActivePlayerとisRoundOverのテストを作成する

  - チップ0のプレイヤーがスキップされることを検証する
  - 全員行動済み・ベット一致でtrueが返ることを検証する
  - 未行動者がいればfalseが返ることを検証する
  - ベット不一致でfalseが返ることを検証する
  - アクティブプレイヤーが1人でtrueが返ることを検証する
  - オールインプレイヤー（チップ0）がアクション不要として扱われることを検証する

  - dealerIndexが-1（初回）の場合にインデックス0から計算が開始されることを検証する

  - Raiseアクションで対象プレイヤーのcurrentBetがcurrentBetの2倍以上になることを検証する
  - applyAction呼び出し後に入力のplayers配列が変更されていないことを検証する
